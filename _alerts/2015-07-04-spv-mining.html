This document is being updated as new information arrives. Last update: 2015-07-04 06:00 UTC

Summary

Some miners are currently generating invalid blocks. Bitcoin Core 0.9.x and earlier will accept these invalid blocks, making them more vulnerable to double spends. This also applies to web wallets from companies that still use 0.9.x or earlier nodes.

Lightweight (SPV) wallets are also at increased risk, and will remain so until all miners have switched back to full validation.

Users of the above listed vulnerable software must either stop accepting transactions or upgrade to Bitcoin Core 0.10.0 or later.

Users of Bitcoin Core 0.10.0 and later are currently unaffected.

Miners: If you are a pool miner, please switch immediately to a pool to that fully validates blocks.
